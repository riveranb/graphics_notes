Depth test + Blending：誰擋誰、怎麼混色

Application Vertex Shader Rasterization Fragment Shader Output Merging
這堂課結束後，你會：
  • 會判斷 Depth test：比深度值，近的寫入、遠的丟掉
  • 知道 z-fighting 從哪來（接第 5 課的深度精度）
  • 看懂 Blending 通用公式：換 Blend Factor 就換效果（Alpha / Additive）
  • 記得透明的兩個規矩：關深度寫入 + 從後往前畫

先看：近的為什麼會擋住遠的

Fragment Shader 算完顏色，還沒寫進畫面。最後的 Output Merging 階段， GPU 先做 Depth test（深度測試）再決定寫不寫。

Depth test：比誰近

每個片段帶著深度值（離鏡頭多遠）。寫入前，跟 Depth Buffer（Z-buffer）裡已記錄的深度比： 

Depth Buffer 目前記錄：z = 0.7（先畫的、較遠的面）

新片段 z = 0.3（較近）→ 比 0.7 近 → 顏色寫入，Depth Buffer 更新成 0.3
新片段 z = 0.9（較遠）→ 比 0.7 遠 → 整個丟掉（被擋住）

預設比較規則是 less（新片段較近才通過）。不透明物體的遮擋就這樣自動搞定，跟繪製順序無關。

對應到 Cocos：material 的 pass 裡這些是 depthStencilState 的參數—— depthTest（開不開測試）、depthFunc（比較規則，預設 less）、 depthWrite（要不要把深度寫回 buffer）。透明物體就是把 depthWrite 關掉的那個開關。
z-fighting：兩面深度幾乎一樣時，精度分不出誰近，畫面上兩面互相搶著閃爍。 第 5 課提過 ÷w 之後近處精度多、遠處精度少，所以 z-fighting 常在遠處。 關鍵是 far / near 的比值別太大：near=1, far=100（比值 100）就比 near=0.01, far=1000（比值十萬）健康得多——near 別貪心設超小。

Blending：透明是「混色」不是「覆蓋」

不透明片段直接覆蓋。但透明片段要跟畫面上已有的顏色混合—— 這就是第 6 課說 Fragment Shader 做不到、要交給這一階段的 Blending。最常見的 Alpha 混合： 

C_out = C_src × α + C_dst × (1 − α)
// C_src = 新片段的顏色    C_dst = 畫面上已有的顏色    α = 透明度 0~1
透明的兩個坑（最常見的透明 bug 根因）：
1. 透明物體要關掉深度寫入，否則會擋掉後面該透出來的東西。
2. 要從後往前排序畫——混色公式不可交換，先後不同、結果不同（引擎會排，但排序是 CPU 端的代價）。

通用公式：換 Factor 就換效果

GPU 只認一條通用式，各種效果都是換兩個權重（Blend Factor）變出來的：

C_out = C_src × F_src  +  C_dst × F_dst
效果F_srcF_dst結果
Alpha Blending α 1 − α 加權平均＝一般透明（玻璃、霧面）
Additive α（或 1） 1 直接疊上去，越疊越亮（火焰、粒子、雷射）

兩者只差一個 F_dst：Alpha 用 1−α 是「跟背景分配比例」； Additive 用 1 是「背景全保留、直接加」——所以會越疊越亮、逼近白色，黑色 (0,0,0) 疊了等於沒疊。 （還有 Screen / Multiply 等組合，完整參數見原始筆記。）

不想付混色的代價？回到 discard

樹葉、鐵絲網這種「邊緣要嘛實心、要嘛全空」的東西，可以不走 Blending—— 用第 6 課的 discard（Alpha Test）：alpha 太低的片段整個丟掉，剩下的當不透明畫。 不用排序、照常寫深度，代價是邊緣沒有漸變、可能鋸齒。

透明處理二選一：要漸變 → Blending（排序＋關深度寫入）；邊緣分明 → discard（省事但硬邊）。
