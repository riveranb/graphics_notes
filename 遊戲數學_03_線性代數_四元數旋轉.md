# 遊戲開發數學
## 電腦圖學 - 線性代數 - 四元數旋轉

四元數（Quaternion）是一種用於表示三維空間中旋轉的數學工具。相較於尤拉角（Euler Angles）和旋轉矩陣（Rotation Matrix），四元數能有效避免萬向鎖（Gimbal Lock）問題，並且在進行旋轉的平滑插值（Spherical Linear Interpolation, Slerp）時表現極佳。

## 核心概念

四元數本質上是一個四維空間中的複數。對於三維空間的旋轉，我們通常使用的是**單位四元數**（Unit Quaternion）。它可以看作是一個純量（Scalar）加上一個三維向量（Vector）的組合：

```math
q = w + xi + yj + zk
```

或者寫成：

```math
q = (w, \vec{v})
```

其中 $w$ 是純量部分，$\vec{v} = (x, y, z)$ 是向量部分。$i, j, k$ 是虛數單位。

### 軸角轉換 (Axis-Angle Representation)

為了讓四元數變得直觀，我們可以將其與繞軸旋轉（Axis-Angle）聯繫起來。假設我們要繞著單位向量 $\vec{u}$ 旋轉 $\theta$ 角度，其對應的四元數公式為：

```math
\begin{aligned}
q &= \left( \cos\left(\frac{\theta}{2}\right), \sin\left(\frac{\theta}{2}\right)\vec{u} \right) \\
&= \cos\left(\frac{\theta}{2}\right) + \sin\left(\frac{\theta}{2}\right)(u_x i + u_y j + u_z k)
\end{aligned}
```

### 點的旋轉公式

若要使用四元數 $q$ 來旋轉三維空間中的一個點（或位置向量） $p$，我們首先需要將 $p$ 擴充為一個純虛四元數 $p = (0, \vec{p})$。對點 $p$ 進行旋轉後的新點 $p'$ 可以透過以下公式計算：

```math
p' = q p q^{-1}
```

*註：對於單位四元數，其反四元數 $q^{-1}$ 等於共軛四元數 $q^*$。*

## Hardware/Pipeline Context

四元數的操作通常發生在 **CPU** 端（Object Level 的 Transform 計算或是 Animation 骨架計算階段）。在遊戲引擎中，物件的旋轉狀態（Rotation）與動畫關鍵幀（Keyframes）絕大部分是以四元數形式儲存與內插。

當渲染準備階段結束，要進入 GPU Pipeline（特別是進入 **Vertex Shader**）進一步針對幾何頂點進行大量座標轉換時，四元數通常會在提交 Draw Call 之前被轉換為 **4x4 旋轉矩陣（Rotation Matrix）**。這是因為現代 GPU 的 ALU 架構對於矩陣乘向量（Matrix-Vector Multiplication）有著高度的最佳化，使用矩陣來處理大量頂點更為高效。

## Implementation

以下是在程式中將軸角（Axis-Angle）轉換為四元數的虛擬碼：

```glsl
// GLSL-like pseudo code
struct Quaternion {
    float w;
    vec3 v;
};

Quaternion AxisAngleToQuaternion(vec3 axis, float angle) {
    Quaternion q;
    float halfAngle = angle * 0.5;
    float s = sin(halfAngle);
    
    q.w = cos(halfAngle);
    q.v = normalize(axis) * s;
    
    return q;
}
```

## 效能優化 (Performance & Optimization)

*   **記憶體佔用 (Memory Footprint)**：四元數只佔用 4 個浮點數（16 Bytes），而 3x3 或 4x4 旋轉矩陣需要 9 到 16 個浮點數。在儲存大量的動畫資料時，四元數能大幅降低記憶體頻寬（Bandwidth）與容量需求。
*   **正規化成本 (Normalization)**：經過多次浮點數運算後，旋轉資料會產生誤差。四元數的正規化（Normalization）運算非常簡單且成本低；相比之下，要對一個帶有誤差的旋轉矩陣重新進行正交化（Orthogonalization），其運算量會大上許多。
*   **平滑插值 (Interpolation)**：四元數的 Slerp（球面線性插值）能以穩定的角速度進行旋轉過渡，不會像尤拉角那樣產生非預期的旋轉路徑。
*   **串接效率 (Concatenation)**：將多次旋轉合併在一起（亦即四元數相乘），相較於 4x4 矩陣相乘需要較少的運算指令（ALU instructions）。
