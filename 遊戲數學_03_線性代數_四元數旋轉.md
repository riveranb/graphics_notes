# 遊戲開發數學
## 電腦圖學 - 線性代數 - 四元數旋轉

四元數（Quaternion）是表示三維旋轉的代數結構，相較於 Euler Angles 可避免 Gimbal Lock，且原生支援 Slerp 插值。

## 核心定義

四元數屬於超複數系統（Hypercomplex Number），由一個純量 $w$ 與一個三維向量 $\vec{v}$ 組成：

```math
q = w + xi + yj + zk = (w,\;\vec{v})
```

其中 $i, j, k$ 滿足：$i^2 = j^2 = k^2 = ijk = -1$。

用於旋轉時，我們使用**單位四元數**（$\|q\| = 1$）。

### 軸角轉換 (Axis-Angle)

繞單位軸 $\hat{u}$ 旋轉角度 $\theta$，對應的單位四元數為：

```math
q = \left( \cos\frac{\theta}{2},\; \sin\frac{\theta}{2}\,\hat{u} \right)
```

### 共軛與逆

```math
\begin{aligned}
q^* &= (w,\;-\vec{v}) \\
q^{-1} &= \frac{q^*}{\|q\|^2}
\end{aligned}
```

對單位四元數，$q^{-1} = q^*$。

### 四元數乘法 (Hamilton Product)

給定 $q_1 = (w_1, \vec{v}_1)$、$q_2 = (w_2, \vec{v}_2)$：

```math
q_1 q_2 = \bigl(w_1 w_2 - \vec{v}_1 \cdot \vec{v}_2,\; w_1\vec{v}_2 + w_2\vec{v}_1 + \vec{v}_1 \times \vec{v}_2\bigr)
```

注意：四元數乘法**不可交換**（$q_1 q_2 \neq q_2 q_1$），但可結合（$(q_1 q_2) q_3 = q_1 (q_2 q_3)$）。

### 點的旋轉

將三維點 $\vec{p}$ 編碼為純四元數 $p = (0, \vec{p})$，旋轉後的結果為：

```math
p' = q\,p\,q^{*}
```

串接多次旋轉 $q_1$ → $q_2$：$q_{\text{total}} = q_2\,q_1$（右到左順序，與矩陣相同）。

## Hardware / Pipeline Context

- **CPU 端**：引擎以四元數儲存物件 Rotation 與動畫 Keyframe，插值（Slerp）同樣在 CPU 完成。
- **GPU 端**：提交 Draw Call 前，四元數轉為 4×4 Matrix 送入 **Vertex Shader**，因 GPU ALU 對 Matrix-Vector Multiply 高度優化。

## Implementation

```glsl
// Axis-Angle → Quaternion (pseudo GLSL)
vec4 axisAngleToQuat(vec3 axis, float angle) {
    float ha = angle * 0.5;
    return vec4(normalize(axis) * sin(ha), cos(ha)); // (xyz=v, w)
}
```

## 效能優化

| 面向 | 說明 |
|---|---|
| **Memory** | 4 floats (16 B) vs. 矩陣 9–16 floats，大量動畫資料下節省顯著 Bandwidth |
| **Normalization** | 四元數正規化為 $q/\|q\|$，成本遠低於矩陣的 Gram-Schmidt 正交化 |
| **Interpolation** | Slerp 保證等角速度、最短路徑旋轉；Euler Angles 無法保證 |
| **Concatenation** | 四元數乘法（16 mul + 12 add）遠少於 4×4 矩陣乘法（64 mul + 48 add） |
