Vertex Shader：你會改的第一種程式碼

Application Vertex Shader Rasterization Fragment Shader Output Merging
這堂課結束後，你會：
  • 知道 Vertex Shader = 每個頂點跑一次的小程式
  • 知道它最重要的工作：算出 gl_Position（頂點在 Clip Space 的位置）
  • 看得懂一支最簡單的 vertex shader 每一行在幹嘛

先看它長怎樣

這就是一支最基本的 Vertex Shader（GLSL）。別怕，我們一行行拆：

attribute vec3 position;          // 頂點位置（來自 Mesh）
attribute vec2 texCoord;          // 頂點 UV
uniform   mat4 modelViewMatrix;   // 全域：Object→Camera 的矩陣
uniform   mat4 projectionMatrix;  // 全域：投影矩陣
varying   vec2 vTexCoord;         // 要傳給 Fragment Shader 的資料

void main() {
    gl_Position = projectionMatrix * modelViewMatrix * vec4(position, 1.0);
    vTexCoord = texCoord;
}

三種變數，看關鍵字就懂

  • attribute — 輸入：這個頂點自己的資料（位置、UV、法線…就是第 2 課的頂點屬性）
  • uniform — 輸入：整批頂點共用的全域值（例如那兩個矩陣）
  • varying — 輸出：算好後傳給 Fragment Shader 的資料

還有一個一定要輸出的內建變數 gl_Position —— 頂點在 Clip Space 的位置。 那一行矩陣乘法，正是第 3 課說的「把頂點從 Object 一路換到 Clip Space」。

重點只有一句：Vertex Shader 決定「每個頂點最後落在畫面哪裡」。模型位置、相機、投影不對，通常來這裡找。

它會跑很多次

模型有幾個頂點，這支程式就跑幾次（而且是 GPU 大量平行同時算）。 所以 vertex shader 要簡單快 —— 每個頂點都要付一次成本。
